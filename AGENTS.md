# AGENTS.md — seootter

## nbdev project — edit notebooks, not lib code

The real source is `nbs/*.ipynb`. Files under `seootter/` are **autogenerated** (marked `AUTOGENERATED! DO NOT EDIT!`). Edit notebooks, then run:

```sh
nbdev_preview   # renders + previews docs locally (Quarto server)
nbdev_export    # re-export lib from notebooks
nbdev_test      # runs notebook tests
```

## Quick start

```sh
uv sync                    # install deps
cp .env.example .env       # set SEEOOTTER_DB_URL, content paths (or just use .env)
seo-otter-add-website ...    # register a site
seo-otter-sync sc-domain:example.com --days 90   # OAuth on first run
seo-otter-report 1           # full site report
```

## CLI commands

All defined in `seootter/cli.py` as `@call_parse` decorated fns, registered as `[project.scripts]` in `pyproject.toml`:
sync, report, audit, rank, trend, top-pages, wins, canob, index-check, index-report, index-refresh, crawl-errors, compare, country-breakdown, schema-check, faq, add-website, list-websites, delete-website.

## Key structure

| Path | Purpose |
|------|---------|
| `nbs/` | Source notebooks (edit here) |
| `seootter/` | Autogenerated lib (do NOT edit directly) |
| `ui/main.py` | FastHTML web app (serves on port 5002) |
| `seootter/cli.py` | All CLI entrypoints |
| `seootter/gsc/` | GSC queries, sync, filters |
| `seootter/report/` | SEO report generation (articles, analysis, generator) |
| `seootter/content/` | Content mapping/parsing helpers |

## Env & config

- `SEEOOTTER_DB_URL` — SQLite DB path (default in `.env`: `nbs/data/seo.db`)
- `token.pickle` — GSC OAuth token (gitignored)
- `client_secrets.json` — GSC OAuth creds in `~/.config/seootter/` (gitignored)
- Content mapper per domain: `~/.config/seootter/mappers/{domain}/mapper.py` exporting `get_url_file_mapping() -> dict[str, str]`

## Dev workflow

- **Python >=3.12**, uses `uv` (lockfile: `uv.lock`)
- CI: `answerdotai/workflows/nbdev3-ci@master` (runs `nbdev_test`)
- Deploy: `answerdotai/workflows/quarto-ghp3@master` (GitHub Pages to `https://abdelkareemkobo.github.io/seootter/`)
- Docs are built by Quarto from notebook exports to `_docs/` (not committed to `main`)

## Style notes

- All CLI output uses `rich` for formatted tables
- GSC queries use `sc-domain:` prefixed site URLs internally
- Content mapping modes: `direct` (Quarto), `slug` (Astro Latin), `limax` (Arabic filenames)
